Тестовое задание.
1. Вход: json c координатами нескольких прямоугольников на плоскости со сторонами, параллельными осям координат. Ввод можно сделать из текстового файла или из консоли  - на выбор.
2. Требуемый функционал: рассчитать общую часть всех прямоугольников (их пересечение).
3. Вывод: прямоугольник в json в том же формате (или «empty») в текстовый файл или в консоль.
 4. Требования к ООП стилю: реализовать класс «прямоугольник» с функциями ввода, вывода, пересечения и другими, если нужно.
Примерный формат json и тестовый пример:
{

    "rects": [{

            "x1": 0.5,

            "x2": 2.5,

            "y1": 1.5,

            "y2": 2.5

        },

        {

            "x1": 1.5,

            "x2": 0.5,

            "y1": 2.5,

            "y2": 0.5

        }

    ]

}
